Privacy Policy

This privacy policy explains how Learn While Walking Upload Automation accesses, uses, stores, and protects information when interacting with Google and YouTube.

Effective date: July 3, 2026

1. Scope

This privacy policy applies only to Learn While Walking Upload Automation, the local macOS desktop application.

The application is operated locally and is not publicly distributed, publicly downloadable, or provided as an online service.

The current application integration supports YouTube. If support for another video platform is added in the future, this privacy policy will be reviewed and updated before that integration is used.

3. Google OAuth permissions

The application uses Google OAuth to obtain authorization from the Google account controlling the relevant YouTube channel.

The application requests only the following OAuth scopes:

  • https://www.googleapis.com/auth/youtube.readonly — used to identify and verify the authenticated YouTube channel.
  • https://www.googleapis.com/auth/youtube.upload — used to upload videos to the authenticated YouTube channel.

The application does not use Google OAuth to create a separate application account or application login.

It does not request access to Gmail, Google Drive, Google Contacts, or other unrelated Google services.

4. Google and YouTube data accessed

Depending on the operation initiated by the user, the application may access or process the following data:

  • The authenticated YouTube channel ID and channel title.
  • Google OAuth authorization credentials, including access and refresh information required to maintain authorized access.
  • Video files selected for upload.
  • Video metadata prepared for upload, such as the title, description, hashtags, category, visibility, and applicable publication information.
  • The YouTube video ID and result information returned after an upload attempt.

The application does not retrieve a general list of the user’s YouTube videos and does not use YouTube data for advertising, profiling, or marketing.

5. How the data is used

Google and YouTube data is used only for the following application functions:

  • Verifying that the authenticated YouTube channel matches the configured destination channel.
  • Uploading a user-selected video and its associated metadata to that channel.
  • Recording the outcome of an upload attempt.
  • Preventing unintended duplicate uploads.
  • Supporting controlled error handling, manual review, and local archival after a verified successful upload.

Google and YouTube data is not used for purposes unrelated to these functions.

6. Local storage

The application runs locally on the operator’s computer. It does not transmit Google or YouTube user data to an application server operated by Niklas Hoppe.

OAuth credentials and language-specific token files are stored locally outside the application source-code directory.

Local upload-result records may contain:

  • Technical upload and attempt identifiers.
  • The destination platform and YouTube channel ID.
  • The source filename, file size, and cryptographic file hash.
  • Upload timestamps and result status.
  • The YouTube video ID when one is available.
  • The visibility requested for the upload.

These records are used for auditability, duplicate protection, error recovery, and archive verification. Complete YouTube API responses are not retained as application result records.

7. Sharing and disclosure

Data required for channel verification and video upload is transmitted directly between the local application and Google or YouTube through the official Google APIs.

The operator does not:

  • Sell Google or YouTube user data.
  • Share Google or YouTube user data with advertising platforms or data brokers.
  • Use the data for personalized advertising.
  • Transfer the data to unrelated third parties.

Information may be disclosed where required by applicable law or a legally binding request.

8. Google API data-use commitment

The use and transfer of information received from Google APIs by Learn While Walking Upload Automation will adhere to the Google API Services User Data Policy , including the Limited Use requirements.

The application’s use of YouTube API Services is also subject to the YouTube API Services Developer Policies and the YouTube API Services Terms of Service .

9. Retention and deletion

OAuth token files remain stored locally until they are deleted, replaced during a deliberate reauthorization, or rendered unusable by revocation.

Upload-result records and archived source files remain under the operator’s local control and are retained until the operator deletes them.

Videos uploaded to YouTube remain in the applicable YouTube account until they are changed or deleted through YouTube or YouTube Studio.

A request concerning locally stored application data may be sent to: niklas.hoppe@learnwhilewalking.ch .

10. Revoking Google access

Google OAuth access may be reviewed or revoked through the Google Account connections page:

Manage connections to your Google Account

The locally stored token file may also be deliberately removed from the operator’s computer.

Revoking authorization or deleting the local token stops future authorized API access. It does not automatically delete videos already uploaded to YouTube or local upload-result and archive records.

11. Security

The application applies technical safeguards intended to reduce unauthorized access and unintended uploads. These safeguards include:

  • Local storage of OAuth credentials outside the application source-code directory.
  • Separate token storage for each configured language channel.
  • Validation of the authenticated YouTube channel ID before an upload.
  • Restricted OAuth scopes.
  • Controlled token refresh and atomic token-file replacement.
  • Duplicate-upload protection and explicit handling of uncertain upload outcomes.

No method of electronic storage or transmission can be guaranteed to be completely secure.
